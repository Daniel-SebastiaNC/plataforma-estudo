Estrutura semântica com HTML5

Aprenda a usar as tags certas para dar significado ao seu conteúdo e melhorar acessibilidade e SEO.

Selecione qualquer trecho para marcar ou adicionar uma observação — tudo fica salvo no Caderno do aluno.

O que é HTML semântico?

HTML semântico significa usar cada elemento pelo seu significado, e não apenas pela aparência. Em vez de encher a página de divs genéricas, você escolhe tags que descrevem o papel do conteúdo: um cabeçalho é um header, a navegação principal é uma nav, e o conteúdo central da página é um main.

Essa escolha tem impacto direto na acessibilidade. Leitores de tela usam essas marcações para permitir que a pessoa navegue direto para o conteúdo principal ou pule entre seções, sem precisar ouvir a página inteira do começo.

Principais elementos

O elemento article representa um bloco de conteúdo independente, que faria sentido sozinho, como um post ou uma notícia. Já o section agrupa conteúdo relacionado dentro de um tema, geralmente com seu próprio título. O aside guarda conteúdo secundário, como uma barra lateral ou uma nota complementar.

Por fim, o footer fecha a página ou uma seção com informações de rodapé, como créditos, contato e links legais. Usar essas tags corretamente deixa o código mais legível e mais fácil de manter por qualquer pessoa da equipe.

Estrutura típica de uma página

Numa página real, esses elementos se combinam formando um esqueleto previsível. No topo fica o header, geralmente com o logotipo e a nav de navegação principal. O miolo da página vive dentro de um único main, que concentra o conteúdo mais importante — aquilo que a pessoa realmente veio consumir.

Dentro do main, você organiza o conteúdo em blocos: um article para cada unidade independente e sections para agrupar temas relacionados. À margem, o aside carrega o que é complementar, como links relacionados ou um resumo. Lá embaixo, o footer encerra a página com as informações de rodapé.

Um exemplo na prática

Repare como a marcação abaixo já comunica a função de cada parte antes mesmo de olhar o CSS. Qualquer pessoa — ou leitor de tela — entende a hierarquia só pela escolha das tags.

<body>
  <header>
    <nav>...</nav>
  </header>

  <main>
    <article>
      <h1>Título do conteúdo</h1>
      <section>...</section>
    </article>
    <aside>...</aside>
  </main>

  <footer>...</footer>
</body>

Erros comuns

O erro mais frequente é a chamada "div-soup": resolver toda a estrutura com divs aninhadas, sem nenhuma tag semântica. O resultado até funciona visualmente, mas o código perde significado e a acessibilidade despenca.

Outro deslize é usar section apenas para criar um espaçamento, sem que aquele bloco tenha de fato um título e um tema próprio — nesses casos, uma div genérica seria mais honesta. Também é comum abusar de header e footer em todo lugar: eles fazem sentido para a página e para seções maiores, não para cada pequeno cartão.

Boas práticas e acessibilidade

Use um único main por documento: ele marca o ponto de partida do conteúdo principal e é justamente para lá que os leitores de tela oferecem um atalho. Mantenha os títulos numa hierarquia coerente, do h1 aos subníveis, sem pular etapas só por causa da aparência.

Elementos como header, nav, main, aside e footer funcionam como landmarks: pontos de referência que permitem navegar a página por regiões. Quando você respeita esse vocabulário, entrega de graça uma experiência muito melhor para quem depende de tecnologias assistivas — e um código mais fácil de manter para o resto da equipe.
